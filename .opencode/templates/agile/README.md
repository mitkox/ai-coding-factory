# Agile Templates

This directory contains templates for enterprise agile development practices.

## Templates

### 1. User Story Template

**File:** `user-story-template.md`

Purpose: Standardized template for creating user stories that follow INVEST criteria.

### 2. Sprint Planning Template

**File:** `sprint-planning-template.md`

Purpose: Template for planning sprint goals, capacity, and user stories.

### 3. Daily Standup Template

**File:** `daily-standup-template.md`

Purpose: Template for conducting daily standup meetings.

### 4. Sprint Review Template

**File:** `sprint-review-template.md`

Purpose: Template for sprint review/demo meetings with stakeholders.

### 5. Sprint Retrospective Template

**File:** `sprint-retrospective-template.md`

Purpose: Template for sprint retrospective meetings to improve team processes.

### 6. Backlog Management Template

**File:** `backlog-management-template.md`

Purpose: Template for managing and prioritizing product backlog.

### 7. Definition of Done

**File:** definition-of-done.md`

Purpose: Standardized checklist for determining when work is complete.

### 8. Definition of Ready

**File:** definition-of-ready.md`

Purpose: Standardized checklist for determining when user stories are ready for sprint.

### 9. Velocity Tracking Template

**File:** velocity-tracking-template.md`

Purpose: Template for tracking team velocity across sprints.

### 10. Release Planning Template

**File:** `release-planning-template.md`

Purpose: Template for planning product releases and milestones.

## Usage

Each template includes:
- Clear structure and format
- Instructions for filling out
- Examples for reference
- Best practices and guidelines

Copy templates to your project and customize based on your team's needs.

## Best Practices

1. **Consistency**: Use same templates across all sprints
2. **Customization**: Adapt templates to your team's needs
3. **Documentation**: Keep historical records for analysis
4. **Collaboration**: Share templates with all team members
5. **Continuous Improvement**: Update templates based on team feedback

## Enterprise Agile Frameworks

This repository supports multiple agile frameworks:

### Scrum
- 2-week sprints
- Defined roles (Scrum Master, Product Owner, Team)
- Ceremonies: Planning, Daily Standup, Review, Retrospective

### Kanban
- Continuous flow
- WIP limits
- Visual board (To Do, In Progress, Review, Done)

### Hybrid Scrum/Kanban
- Scrum for planning and ceremonies
- Kanban for visual work tracking
- Flexible iteration boundaries

## Metrics and KPIs

Track the following metrics to measure agile maturity:

- **Velocity**: Story points completed per sprint
- **Cycle Time**: Time from story start to finish
- **Lead Time**: Time from story creation to finish
- **Defect Rate**: Bugs found post-release
- **Sprint Success Rate**: % of sprints meeting goals
- **Team Satisfaction**: Regular survey metrics
